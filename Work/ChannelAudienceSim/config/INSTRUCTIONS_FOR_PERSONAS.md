# Как писать инструкции для персон

Каждая персона = отдельный юзербот (отдельный аккаунт Telegram). Сообщения генерирует **только LLM** по полю `instruction` из `personas.yaml`.

## Что задаёт инструкция

- **Личность:** кто этот человек (возраст, сфера, привычки).
- **Психология:** как реагирует на посты — поддерживает, сомневается, шутит, задаёт вопросы.
- **История/контекст:** откуда пришёл в канал, что ему интересно, что раздражает.
- **Стиль:** длина реплик, грамотность, сленг, эмодзи, тон (тёплый / нейтральный / колкий).

Модель получает твой текст как системный промпт и генерирует один комментарий или ответ в треде **от имени этой персоны**, без подписи (аккаунт уже другой).

## Формат в YAML

```yaml
- id: unique_id
  name: Имя (как в голове, для отладки)
  session: имя_файла_сессии   # без .session
  probability: 0.7           # вероятность написать под постом (0–1)
  delay_range: [60, 300]     # задержка в секундах [мин, макс] + джиттер
  reply_probability: 0.6     # вероятность ответить в треде (0–1)
  instruction: |
    Текст инструкции в несколько строк.
    Кто он, как пишет, как реагирует.
```

## Примеры идей для instruction

- **Подписчик-энтузиаст:** давно читает канал, благодарит, задаёт уточняющие вопросы, иногда делится своим опытом по теме.
- **Скептик:** задаёт неудобные вопросы, указывает на упущения, но без троллинга — конструктивно.
- **Тихоня:** пишет редко, коротко — «согласен», «полезно», «жду продолжения».
- **Юморист:** реагирует с шуткой или иронией, может кинуть мем/эмодзи, не разводит длинные тексты.
- **Эксперт:** иногда добавляет факт или уточнение по теме поста, без менторства.

Чем конкретнее инструкция (возраст, профессия, как формулирует мысли), тем стабильнее «живая» имитация канала.
