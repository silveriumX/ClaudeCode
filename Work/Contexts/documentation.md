# Контекст: Документация (documentation)

**Фокус:** Documentation/, API, Cursor, инструкции, единый источник правды (canonical/REF).

## Что соблюдать

- **file-indexing-storage.mdc** — куда класть новые файлы, обновление INDEX.md.
- Навигация: `NAVIGATION.md`, `Documentation/Cursor/INDEX.md`, `Projects/INDEX.md`, `Scripts/INDEX.md`.

## Ключевые папки

- `Documentation/` — API, Cursor, GitHub, Team, Instructions.
- `Documentation/Cursor/` — правила Cursor, гайды, контекст.
- Новые доки — в подходящую подпапку и обновить индекс.

## Canonical и ссылки (антигаллюцинации)

- **Один канонический источник** на утверждение/метрику — помечать в документе: `[CANONICAL]`.
- Остальные упоминания — ссылка: `[REF: путь/файл.md#раздел]`.
- Не дублировать формулировки; при изменении каноника — обновить все `[REF:]`.
- Плейсхолдеры: `[PLACEHOLDER: владелец]`; даты «Last Updated» где уместно.

## В этом контексте

- Предлагать структуру документа и куда его положить.
- Предлагать обновить INDEX и NAVIGATION при новых файлах.
- При запросе «документировать» — уточнять каноник и ссылки.
