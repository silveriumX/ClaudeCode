# Контекст: Проекты (projects)

**Фокус:** Projects/, боты и приложения, общий деплой, структура проектов.

## Что соблюдать

- **file-indexing-storage.mdc** — новые файлы в нужный проект, обновлять `Projects/INDEX.md`.
- **development-workflow-quality.mdc** — при разработке: спецификация → тесты → реализация → проверка → коммит.
- **python-standards.mdc** — для Python-кода в проектах.

## Ключевые пути

- `Projects/` — FinanceBot, VoiceBot, CreatorBot, ChatManager, ServerManager и др.
- `Projects/INDEX.md` — индекс проектов.
- Деплой: по проекту (FinanceBot — см. finance-bot-practices; другие — свои гайды в проекте).

## В этом контексте

- Отвечать в масштабе «проектов»: какой проект, куда положить код/документ.
- Ссылаться на README и доки внутри конкретного проекта при вопросах по нему.
- Не смешивать контекст нескольких ботов без уточнения (например, FinanceBot vs VoiceBot).
