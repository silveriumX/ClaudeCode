# Fireflies: интеграция и лучшие практики по транскрипции встреч

Настройка Cursor (rules, skills, MCP) для автоматической работы с транскриптами встреч через Fireflies.ai — без лишнего участия пользователя.

---

## 1. Интеграция Fireflies с Cursor

### 1.1 MCP-сервер Fireflies

В проекте уже добавлен MCP Fireflies в `.cursor/mcp.json`:

```json
"fireflies": {
  "command": "npx",
  "args": [
    "-y",
    "mcp-remote",
    "https://api.fireflies.ai/mcp",
    "--header",
    "Authorization: Bearer YOUR_FIREFLIES_API_KEY"
  ],
  "env": {}
}
```

**Что сделать вам:**

1. Зайти на [app.fireflies.ai](https://app.fireflies.ai) → **Integrations** → **Fireflies API** (или **Settings** → **Developer Settings**).
2. Скопировать **API key**.
3. В `.cursor/mcp.json` заменить `YOUR_FIREFLIES_API_KEY` на свой ключ.
4. Перезапустить Cursor (или перезагрузить MCP), чтобы сервер Fireflies подключился.

**Безопасность:** не коммитить `mcp.json` с реальным ключом. Можно держать ключ в локальной копии файла или в переменной окружения и подставлять его вручную.

### 1.2 Доступные инструменты MCP (кратко)

| Инструмент | Назначение |
|------------|------------|
| `fireflies_search` | Поиск по встречам (грамматика: даты, участники, ключевые слова). |
| `fireflies_get_transcripts` | Список встреч по фильтрам (даты, участники, организаторы, лимит до 50). |
| `fireflies_get_summary` | Саммари одной встречи: экшены, ключевые слова, обзор (без полного текста). |
| `fireflies_get_transcript` | Полный транскрипт одной встречи (предложения, спикеры, таймкоды). |
| `fireflies_fetch` | Всё по одной встрече: транскрипт + саммари + метаданные. |
| `fireflies_get_user` | Данные пользователя/аккаунта. |
| `fireflies_get_usergroups` | Группы/команды. |
| `fireflies_get_user_contacts` | Контакты по последним встречам. |

Подробно: [Fireflies MCP Tools](https://docs.fireflies.ai/mcp-tools/overview).

## 2. Правила и скилл в Cursor

### 2.1 Rule: meeting-transcripts

**Файл:** `.cursor/rules/meeting-transcripts.mdc`

- **Когда применяется:** при работе с файлами в `Work/Задачи/`, с документацией Fireflies или с файлами встреч.
- **Что задаёт:**
  - порядок вызовов MCP (сначала поиск/список, потом summary/transcript/fetch);
  - единый формат ответа (саммари, экшены, темы);
  - интеграцию с планом и `АКТУАЛЬНЫЕ_ЗАДАЧИ.md` при добавлении экшенов из встреч.

### 2.2 Skill: fireflies-meetings

**Файл:** `.cursor/skills/fireflies-meetings/SKILL.md`

- **Когда использовать:** запросы про Fireflies, транскрипцию встреч, саммари, экшены, поиск по встречам, автоматическое добавление задач в план.
- **Что делает агент:** по инструкции из скилла выбирает нужные MCP-инструменты, выдаёт результат в одном формате и при необходимости обновляет план/задачи.

В итоге процесс идёт «почти без вашего участия»: вы формулируете запрос (например, «саммари последней встречи с X и добавь экшены в план»), агент сам дергает Fireflies и план.

## 3. Лучшие практики (как у лидеров рынка)

Условные «пресеты» по типу запроса — так обычно строят пайплайны в Otter, Fireflies, Fathom и аналогах.

### 3.1 Точность и платформы

- **Fireflies:** 69 языков, Zoom / Google Meet / Teams; хорошая точность, интеграции с CRM.
- **Типичный пресет:** один инструмент для записи (Fireflies), один источник правды для транскриптов (Fireflies MCP в Cursor), одна система задач (план в `Work/Задачи/`).

Рекомендация: не дублировать транскрипцию в нескольких сервисах; выбрать один (здесь — Fireflies) и всё вести через него и Cursor.

### 3.2 Пресеты запросов (что говорить агенту)

Эти формулировки хорошо работают с правилом и скиллом:

| Цель | Пример запроса |
|------|-----------------|
| Краткое саммари встречи | «Саммари встречи [название/дата]» / «Кратко что было на созвоне с X». |
| Только экшены | «Какие экшены с встречи [дата/название]?» / «Вытащи задачи с созвона». |
| Добавить экшены в план | «Добавь экшены с последней встречи с X в план на сегодня». |
| Анализ по периоду | «Объекции в продажах за эту неделю» / «Фидбек по продукту за месяц». |
| По участнику | «Что поручили мне на прошлой неделе по встречам?» / «Встречи с [имя/email]». |
| Тренды | «Как менялось настроение клиентов в звонках за квартал?» (агрегация по нескольким встречам). |

Агент по правилу и скиллу сам выберет `fireflies_search` / `fireflies_get_transcripts`, затем `fireflies_get_summary` или `fireflies_fetch` и при необходимости обновит план.

### 3.3 Экономия токенов

- Сначала использовать **summary** (`fireflies_get_summary`), а полный транскрипт (`fireflies_get_transcript` / `fireflies_fetch`) — только если нужен дословный разбор.
- В MCP при возможности использовать формат **toon** (по умолчанию) для последующей обработки и **text** для человекочитаемого ответа в чат.

### 3.4 Единый формат вывода (как в правиле)

Для любой встречи агент стремится выдавать:

1. **Саммари** — 2–4 предложения.
2. **Action items** — список с ответственным, если есть.
3. **Ключевые темы/решения.**
4. При необходимости — дата встречи / ссылка.

Так проще вручную проверять и вставлять в отчёты или в задачи.

## 4. Автоматизация «без участия»

Что уже сделано, чтобы процесс шёл легко и автоматично:

1. **MCP Fireflies** в Cursor — один раз подставить API key в `mcp.json`.
2. **Rule `meeting-transcripts`** — задаёт порядок вызовов MCP и формат ответа при работе с задачами/встречами.
3. **Skill `fireflies-meetings`** — подключается по запросам про встречи и ведёт весь сценарий (поиск → саммари/транскрипт → при необходимости обновление плана).
4. **Связь с планом и АКТУАЛЬНЫЕ_ЗАДАЧИ** — экшены из встреч добавляются в план и синхронизируются по `actual-tasks-sync.mdc`.

Вам остаётся:

- Включить Fireflies на своих встречах (Zoom/Meet/Teams).
- В Cursor задавать запросы вида выше — агент сам вызовет Fireflies и при необходимости обновит задачи.
