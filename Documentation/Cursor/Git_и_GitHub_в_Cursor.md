*Создано: 17.01.2026*

## 📍 Где находятся Git-инструменты в Cursor?

### 1️⃣ **Source Control (Панель управления версиями)**
```
Горячая клавиша: Ctrl + Shift + G
Или: Кликнуть на иконку "ветки" в левой панели (3-я иконка сверху)
```

### 2️⃣ **Git Graph (Граф коммитов)**
```
Где: Обычно внизу редактора или в отдельной вкладке
Показывает: Визуальную историю всех изменений
```

### 3️⃣ **Статус-бар (Нижняя строка)**
```
Показывает: master*  ☁  ↑2 ↓1
          └─┬─┘│   │   │  │
            │  │   │   │  └─ Получить 1 коммит
            │  │   │   └──── Отправить 2 коммита
            │  │   └──────── Синхронизация включена
            │  └──────────── Есть несохраненные изменения
            └─────────────── Текущая ветка
```

## 🎯 Git Graph - Панель визуализации

### **Иконки верхней панели:**

```
┌──────────────────────────────────────────────────────────┐
│  ⚡Auto  ⊙  ↓  ↓  ☁↑  ↻                       [master]   │
│   │      │   │   │   │    │                              │
│   │      │   │   │   │    └─ Обновить граф              │
│   │      │   │   │   └────── PUSH (отправить)           │
│   │      │   │   └────────── PULL (скачать)             │
│   │      │   └────────────── FETCH (проверить)          │
│   │      └────────────────── Все ветки                  │
│   └───────────────────────── Автопоказ текущей ветки    │
└──────────────────────────────────────────────────────────┘
```

### **Что означают символы:**

| Иконка | Название | Действие | Когда использовать |
|--------|----------|----------|-------------------|
| ⚡ **Auto** | Авто-режим | Показывает текущую ветку | По умолчанию включено |
| ⊙ | Все ветки | Показать все ветки проекта | Когда работаете с feature-ветками |
| ↓ **Fetch** | Получить инфо | Проверяет новые коммиты на GitHub | Перед началом работы |
| ↓ **Pull** | Скачать | Скачивает изменения с GitHub | Работа в команде / с других ПК |
| ☁↑ **Push** | Отправить | Отправляет ваши коммиты на GitHub | После каждого коммита |
| ↻ | Обновить | Перезагрузить граф | Если что-то не обновилось |

## 🔄 Полный цикл работы с Git

### **Шаг 1: Редактирование файлов**
```
📝 Вы редактируете файлы
   ↓
🟡 В статус-баре появляется: master*
   (звездочка = есть изменения)
```

### **Шаг 2: Просмотр изменений**
```
1. Нажмите: Ctrl + Shift + G
2. Увидите список измененных файлов:

   SOURCE CONTROL
   ├─ 📝 Changes (3)
   │  ├─ M  План на 18.01.2026.md
   │  ├─ M  Задачи.md
   │  └─ A  новый_файл.md

   M = Modified (изменен)
   A = Added (добавлен)
   D = Deleted (удален)
```

### **Шаг 3: Подготовка коммита (Stage)**
```
✅ Нажмите "+" возле каждого файла
   ИЛИ
✅ Кнопка "Stage All Changes" (добавить все)

Файлы переместятся в раздел "Staged Changes"
```

### **Шаг 4: Создание коммита**
```
1. В поле "Message" вверху напишите описание:
   ✅ "Обновлен план на 18.01.2026"
   ❌ "фикс" (слишком кратко)

2. Нажмите: ✓ Commit (или Ctrl + Enter)
```

### **Шаг 5: Отправка на GitHub (Push)**
```
Вариант А: В Git Graph нажмите ☁↑ (Push)
Вариант Б: В Source Control нажмите "⋯" → Push
Вариант В: В статус-баре кликните на ☁

Готово! ✨ Изменения теперь на GitHub
```

## 🎨 Визуальные подсказки

### **Граф коммитов - как читать:**

```
🔵─────────────────────── master (HEAD)
│  "Добавлен план на 18.01"
│
🔵───────────────────────
│  "Обновлены задачи"
│
🔵───────────────────────
   "Initial commit"

HEAD = текущая позиция
master = ветка master
🔵 = коммит (точка в истории)
```

### **Статус-бар - расшифровка:**

```
Вариант 1: master*
└─ Есть несохраненные изменения, нужен коммит

Вариант 2: master ☁ ↑3
└─ Есть 3 коммита для отправки на GitHub

Вариант 3: master ☁ ↓2
└─ На GitHub есть 2 новых коммита для скачивания

Вариант 4: master ☁ ↑1 ↓1
└─ Нужно отправить 1 и получить 1 коммит
```

## ⚠️ Типичные ситуации и решения

### **1. Звездочка не пропадает после коммита**
```
Проблема: master* (звездочка осталась)
Причина: Не все файлы добавлены в коммит

Решение:
1. Ctrl + Shift + G
2. Проверьте раздел "Changes"
3. Добавьте оставшиеся файлы (Stage All)
4. Сделайте новый коммит
```

### **2. Кнопка показывает "Publish Branch"**
```
Ситуация: Ветка существует локально, но не на GitHub

Решение:
1. Нажмите "Publish Branch"
2. Ветка автоматически создастся на GitHub
3. В следующий раз будет обычный Push
```

### **3. Конфликт при Pull**
```
Ошибка: "Merge conflict in файл.md"

Причина: Файл изменен и локально, и на GitHub

Решение:
1. Откройте файл с конфликтом
2. Увидите метки:
   <<<<<<< HEAD
   Ваша версия
   =======
   Версия с GitHub
   >>>>>>> origin/master

3. Выберите нужный вариант
4. Удалите метки (<<<<, ====, >>>>)
5. Сохраните и закоммитьте
```

### **4. Нужно отменить последний коммит**
```
Решение:
1. В Git Graph кликните правой кнопкой на коммите
2. Выберите "Reset current branch to this commit"
3. Выберите режим:
   - Soft: сохранит изменения
   - Hard: удалит изменения (ОПАСНО!)
```

## 🎯 Быстрые команды (Terminal)

Если предпочитаете терминал, вот основные команды:

```bash
# Проверить статус
git status

# Добавить все изменения
git add .

# Создать коммит
git commit -m "Описание изменений"

# Отправить на GitHub
git push

# Получить изменения
git pull

# Посмотреть историю
git log --oneline --graph

# Проверить удаленный репозиторий
git remote -v
```

## 🎓 Лучшие практики

### ✅ **Хорошие привычки:**
1. **Коммитьте часто** - маленькие коммиты лучше больших
2. **Пишите понятные сообщения** - "Добавлен план на 18.01" вместо "фикс"
3. **Делайте Pull перед началом работы** - избежите конфликтов
4. **Проверяйте изменения** - перед коммитом посмотрите, что меняется
5. **Push регулярно** - не копите коммиты локально

### ❌ **Чего избегать:**
1. Не коммитьте временные файлы
2. Не используйте `git push --force` без понимания
3. Не храните пароли и ключи в репозитории
4. Не работайте с одним файлом с разных устройств без синхронизации

## 🆘 Частые вопросы

**Q: Как посмотреть, что изменилось в файле?**
```
A: В Source Control (Ctrl+Shift+G) кликните на файл
   Откроется diff с подсветкой изменений:
   - Красный = удалено
   + Зеленый = добавлено
```

**Q: Как вернуться к старой версии файла?**
```
A: 1. Откройте Git Graph
   2. Найдите нужный коммит
   3. ПКМ → "Checkout commit"
   Или через терминал: git checkout <hash> -- путь/к/файлу
```

**Q: Что делать, если случайно закоммитил лишнее?**
```
A: Если НЕ делали Push:
   - Git Graph → ПКМ на коммите → Reset → Soft

   Если УЖЕ сделали Push:
   - Создайте новый коммит с исправлением
   (НЕ используйте force push!)
```
