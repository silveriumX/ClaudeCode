# Git для новичка: твой проект, Cursor и GitHub

*Персональная шпаргалка под твою текущую настройку. Обновлено: 12.02.2026*

---

## 1. Где что лежит: два места вместо одного

У тебя есть **одно и то же хранилище кода в двух местах**:

| Место | Путь / адрес | Что это |
|-------|----------------|--------|
| **У тебя на диске** | `C:\Users\Admin\Documents\workspaces for ai\Cursor` | Папка проекта. Все файлы, которые ты открываешь в Cursor, лежат здесь. |
| **На GitHub** | https://github.com/silveriumX/Cloud_Cursor | Копия репозитория в облаке. Та же структура папок и файлов, но версия с последнего Push. |

- **Редактируешь ты всегда только локальную папку** (на диске).
- **GitHub** — это бэкап и синхронизация: ты отправляешь туда изменения (Push) и при необходимости забираешь чужие или свои с другого ПК (Pull).

Файлы **не открываются с GitHub напрямую**. В Cursor открыта именно локальная папка; Git только синхронизирует её с GitHub.

## 2. Текущая настройка (твой проект)

- **Репозиторий на GitHub:** `silveriumX/Cloud_Cursor`
- **Ветка:** `main` (и локально, и на GitHub)
- **Связь:** одна удалённая ветка `origin` → `https://github.com/silveriumX/Cloud_Cursor.git`

То есть: один репозиторий, одна ветка `main`, один remote с именем `origin`.

## 3. Где в Cursor смотреть и куда тыкать

### 3.1 Панель Source Control (управление Git)

- **Как открыть:** `Ctrl + Shift + G` или иконка ветки в **левой панели** (третья сверху).
- **Что там:**
  - **CHANGES** — список изменённых и новых файлов (то, что ещё не закоммичено).
  - **Поле Message** — сообщение для следующего коммита.
  - **Кнопка Commit** (и выпадающее меню: Commit, Commit & Push, Commit & Sync) — сохранить изменения в историю и при желании отправить на GitHub.

Здесь ты делаешь: смотришь изменения → добавляешь файлы в коммит (Stage) → пишешь Message → жмёшь Commit или Commit & Push.

### 3.2 Нижняя строка (статус-бар)

Внизу окна Cursor показывается что-то вроде:

- `main` — текущая ветка.
- `main*` — есть несохранённые изменения в файлах или не закоммиченные изменения.
- `↑ 10` — у тебя 10 коммитов, которые ещё не отправлены на GitHub (твоя текущая ситуация).
- `↓ 2` — на GitHub есть 2 коммита, которых у тебя нет (нужен Pull).

Клик по названию ветки / облаку часто открывает меню Pull / Push / Sync.

### 3.3 Где физически файлы

- Все файлы, которые ты видишь в проводнике Cursor (слева), — это файлы в папке
  `C:\Users\Admin\Documents\workspaces for ai\Cursor`.
- Открываешь файл — открывается с диска. Сохраняешь — пишется на диск.
- Git лишь **отслеживает** изменения в этой папке и по твоей команде (Commit → Push) отправляет их на GitHub.

## 4. Что значит, что у тебя сейчас

Кратко по текущему состоянию:

- **`main...origin/main [ahead 10]`**
  У тебя есть **10 коммитов**, которые уже сделаны локально, но ещё **не отправлены** на GitHub. После Push они появятся в репозитории.

- **Буквы в списке изменений:**
  - **M** (Modified) — файл уже в Git, ты его изменил.
  - **D** (Deleted) — файл был в Git, ты его удалил.
  - **??** (Untracked) — новый файл/папка, Git их ещё не отслеживает.

Часть файлов (например `.env`, папки из `.gitignore`) в списке не появятся — Git их игнорирует и не коммитит (так и задумано).

## 5. Что делать по шагам (практика)

### Шаг 0: Открыть Source Control

Нажми **Ctrl + Shift + G**. Слева откроется панель с изменениями.

### Шаг 1: Решить, что именно коммитить

- Можно добавлять файлы по одному: наведи на файл → нажми **"+"** (Stage).
- Или добавить все изменения: кнопка **"Stage All Changes"** (или через меню "...").

Не бойся: в коммит **не попадут** файлы из `.gitignore` (например `.env`, ключи, папки типа `Личное/` и т.п.). То, что не попало в список Changes, Git не трогает.

### Шаг 2: Написать сообщение коммита

В поле **Message** вверху панели Source Control напиши коротко, что сделано, например:

- `docs: обновление документации и планов`
- `chore: добавлены новые скрипты и проекты`

Так потом проще понять историю.

### Шаг 3: Сделать коммит и отправить на GitHub

- Нажми на **стрелку** рядом с кнопкой **Commit** и выбери **"Commit & Push"**.
- Или сначала **Commit**, потом в меню "..." в Source Control выбери **Push**.

После этого твои 10 локальных коммитов + новый коммит уедут на GitHub, и внизу исчезнет "↑ 10" (или число станет меньше).

### Шаг 4: Проверить на GitHub

Открой в браузере: https://github.com/silveriumX/Cloud_Cursor
Там должны появиться последние коммиты и актуальные файлы.

## 6. Коротко: куда тыкать и зачем

| Действие | Куда тыкать |
|----------|-------------|
| Посмотреть все изменения | **Ctrl + Shift + G** → смотреть список в CHANGES. |
| Добавить файл в коммит | В Source Control: **"+"** у файла или **Stage All Changes**. |
| Сохранить изменения в историю (только локально) | Написать **Message** → кнопка **Commit**. |
| Сохранить и отправить на GitHub | **Message** → выпадающее меню у Commit → **Commit & Push**. |
| Подтянуть изменения с GitHub | В Source Control: меню **"..."** → **Pull**, или в статус-баре (если есть ↓). |
| Открыть репозиторий в браузере | Перейти по ссылке: https://github.com/silveriumX/Cloud_Cursor |

## 7. Важно помнить

1. **Файлы живут у тебя на диске** в папке `workspaces for ai\Cursor`. GitHub — копия, которую ты обновляешь через Push.
2. **Commit** — это сохранение "снимка" изменений в локальную историю. **Push** — отправка этих снимков на GitHub.
3. То, что в **.gitignore**, не коммитится и не уедет на GitHub (токены, .env, личные папки и т.д.).
4. Если работаешь один и с одного ПК — чаще всего достаточно: **Stage → Message → Commit & Push**.
