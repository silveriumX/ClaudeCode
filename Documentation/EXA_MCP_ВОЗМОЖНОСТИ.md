# EXA MCP — возможности и использование

**Статус:** подключён в `.cursor/mcp.json`
**Ключ:** хранится в `.env` (переменная `EXA_API_KEY`), в репозиторий не попадает

---

## Что такое EXA MCP

EXA MCP подключает Cursor к семантическому поиску Exa: веб-поиск, поиск по коду, исследование компаний и людей. Поиск по смыслу, а не только по ключевым словам.

---

## Инструменты (включены по умолчанию)

| Инструмент | Описание |
|------------|----------|
| **web_search_exa** | Поиск по вебу с возвратом чистого контента (без лишней вёрстки) |
| **get_code_context_exa** | Поиск примеров кода и документации (GitHub, Stack Overflow и др.) |
| **company_research_exa** | Исследование компаний по их сайтам |

## Дополнительные инструменты (включить при необходимости)

Их можно включить, используя удалённый URL с параметром `tools`:

```
https://mcp.exa.ai/mcp?tools=web_search_exa,web_search_advanced_exa,get_code_context_exa,deep_search_exa,crawling_exa,company_research_exa,people_search_exa,deep_researcher_start,deep_researcher_check
```

| Инструмент | Описание |
|------------|----------|
| **web_search_advanced_exa** | Расширенный поиск с фильтрами |
| **deep_search_exa** | Глубокий поиск с расширением запроса |
| **crawling_exa** | Получение контента по конкретному URL |
| **people_search_exa** | Поиск людей и их профилей в соцсетях |
| **deep_researcher_start** | Запуск AI-исследователя |
| **deep_researcher_check** | Проверка статуса исследования и получение отчёта |

## Как использовать в Cursor

### 1. Перезапуск Cursor

После добавления EXA в `mcp.json` перезапустите Cursor, чтобы подтянуть новый MCP-сервер.

### 2. Примеры запросов

- **Веб-поиск:**
  *«Найди последние новости про AI-агентов и выдели главные тренды»*

- **Поиск кода:**
  *«Найди примеры на Python для OAuth 2.0»*

- **Исследование компании:**
  *«Изучи Stripe: продукты и последние новости»*

- **Глубокий ресерч (если включён deep_researcher):**
  *«Сделай развёрнутый отчёт по квантовым стартапам»*

### 3. Связь с Research System

Skill **research-system** использует EXA как основной источник (приоритет 1) для:

- сбора информации по теме;
- ресерча по людям, компаниям, технологиям;
- подготовки структурированных отчётов.

Фразы для запуска: *«Исследуй тему…»*, *«Проведи ресерч…»*, *«Найди информацию о…»*.

## Где хранится ключ (безопасность)

- **Ключ:** в файле `.env` в корне workspace: `EXA_API_KEY=...`
- **В mcp.json:** используется ссылка `"EXA_API_KEY": "${env:EXA_API_KEY}"` — самого ключа в конфиге нет, его можно коммитить.
- **`.env`** уже в `.gitignore` — в репозиторий не попадёт.
- **Если EXA MCP не подхватывает ключ:** Cursor может не загружать `.env` при старте. Задайте `EXA_API_KEY` в переменных окружения системы: *Параметры системы → Дополнительные параметры → Переменные среды → Переменные среды пользователя → Создать*.
