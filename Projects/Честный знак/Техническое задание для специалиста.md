# Техническое задание для специалиста по Честному знаку

## О компании и контексте

Мы продаем биологически активные добавки (БАДы) на маркетплейсе Wildberries. В ассортименте: жиросжигатели, витамин D3 и другие добавки.

Несколько месяцев назад начали работать с системой маркировки **Честный знак**, которая обязательна для категории БАД. Так как опыт работы с маркировкой небольшой, в процессе возникло несколько проблем, которые требуют профессионального решения.

## Цель работы

Нам нужен внешний специалист, который:
1. Разберется в текущей ситуации с маркировкой
2. Оценит масштаб проблем
3. Разработает план решения
4. Реализует решения самостоятельно (с согласованием этапов)
5. Настроит правильные процессы на будущее
6. При успешной работе — возможно продолжение на постоянной основе

---

## Доступы для работы

Для анализа и решения проблем будут предоставлены:

1. **Личный кабинет Честный знак** — для работы с КИЗами, УПД, анализа цепочки документооборота
2. **Личный кабинет Wildberries** — для выгрузки отчетов по продажам, сборочных заданий, данных по ФБС/ФБО
3. **Контакт бухгалтера поставщика** — для пересогласования УПД, уточнения информации по отгрузкам

## Проблемы, которые нужно решить

### Проблема №1: ФБС — Продажа товаров без ввода в оборот

**Что произошло:**

Поставщик отгрузил нам товар. Wildberries начал продавать этот товар по модели ФБС (забирал под конкретные заказы с нашего склада). Но мы не успели вовремя принять универсальный передаточный документ (УПД) от поставщика в системе Честный знак.

**Последствия:**

- Товар фактически продан конечным покупателям
- КИЗы (коды маркировки) до сих пор числятся за нашим юридическим лицом в системе Честный знак
- Товар не был введен в оборот, хотя уже выбыл из реального оборота

**Усугубляющий фактор:**

По части таких продаж КИЗы не были указаны в сборочных заданиях Wildberries. Из-за этого невозможно точно определить, какие именно коды были проданы и когда.

**Предлагаемое решение:**

1. Выгрузить продажи по проблемным артикулам из личного кабинета WB
2. Сопоставить даты продаж и цены
3. На основе этого анализа определить, какие КИЗы должны быть выведены из оборота
4. Вывести эти КИЗы из оборота вручную в системе Честный знак

**Что нужно от специалиста:**

- Проанализировать ситуацию в обоих кабинетах
- Уточнить оптимальный способ решения проблемы
- Реализовать вывод КИЗов из оборота

### Проблема №2: ФБС — Ручной вывод из оборота принятых товаров

**Что произошло:**

Есть товары, которые были приняты правильно:
1. Поставщик отправил УПД
2. Мы приняли и подписали УПД
3. КИЗы корректно отобразились в нашем кабинете Честный знак
4. Затем товары были проданы по модели ФБС
5. КИЗы были указаны в сборочных заданиях WB

**Требование Wildberries:**

При продажах по модели ФБС продавец должен **самостоятельно**:
- Выводить КИЗы из оборота при продаже товара
- Возвращать КИЗы в оборот при возврате товара от покупателя

**Задача:**

Настроить процесс ручного управления КИЗами:
- Определить, какие товары проданы (на основе отчетов WB)
- Вывести соответствующие КИЗы из оборота
- Настроить регулярный процесс для будущих продаж и возвратов

**Что нужно от специалиста:**

- Разобраться, какие КИЗы должны быть выведены из оборота
- Реализовать вывод
- Разработать процесс на будущее (чек-лист, инструкция)

### Проблема №3: ФБС — Неправильные КИЗы в сборочных заданиях

**Что произошло:**

У нас есть два типа товаров:
- **Одиночный товар** — банка жиросжигателя (1 шт.)
- **Комплект** — набор из 4 единиц (включает ту же банку жиросжигателя + другие позиции)

**Ошибка:**

При формировании сборочных заданий для одиночного товара случайно указывали КИЗы комплекта (набора из 4 единиц) вместо КИЗов одиночной банки.

**Последствия:**

- В системе Wildberries отображается, что мы продали комплект
- Фактически отправили покупателю только одиночную банку
- В системе Честный знак неправильные КИЗы выведены из оборота (от комплекта вместо одиночного товара)

**Задача:**

1. Выявить все случаи неправильного указания КИЗов
2. Понять масштаб проблемы (сколько таких ошибок)
3. Исправить ситуацию в системе Честный знак (вернуть КИЗы комплектов в оборот, вывести правильные КИЗы одиночных товаров)

**Что нужно от специалиста:**

- Найти все проблемные сборочные задания
- Определить, какие КИЗы были выведены ошибочно
- Скорректировать данные в системе Честный знак

### Проблема №4: ФБО — Разрыв цепочки документооборота

**Что произошло:**

При работе по модели ФБО (товар отгружается на склады Wildberries заранее):

1. Мы отгрузили товар на склады Wildberries
2. Wildberries принял товар к себе на склад
3. Мы передали УПД от нашего юрлица в адрес ООО РВБ (юрлицо Wildberries)
4. Wildberries продал товар конечным покупателям

**Проблема:**

Изначально мы **не приняли УПД от поставщика**. То есть КИЗы не были введены в оборот на наше юрлицо.

**Неправильная цепочка документов:**

```
Поставщик (производитель)
    ↓ (УПД не принят!)
Наше юрлицо (мы создали УПД, но товар к нам не приходил по документам!)
    ↓ (УПД передан)
ООО РВБ (Wildberries)
    ↓
Конечный покупатель
```

**Правильная цепочка должна быть:**

```
Поставщик (производитель)
    ↓ (УПД принят)
Наше юрлицо
    ↓ (УПД передан)
ООО РВБ (Wildberries)
    ↓
Конечный покупатель
```

**Задача:**

1. Выявить все такие случаи
2. Связаться с бухгалтером поставщика
3. Запросить повторное формирование УПД от поставщика к нашему юрлицу (если возможно)
4. Восстановить правильную цепочку документооборота

**Что нужно от специалиста:**

- Проанализировать цепочки документов в Честном знаке
- Определить масштаб проблемы
- Пообщаться с поставщиком (через нас или напрямую)
- Реализовать пересогласование документов

## Формат работы

**Что важно:**

- **Полная самостоятельность** — мы ожидаем не консультации, а выполнения работы. Вы должны сами войти в кабинеты, разобраться, реализовать решения.
- **Согласование этапов** — перед критичными действиями (пересогласование УПД, массовый вывод КИЗов) показываете план и получаете подтверждение.
- **Прозрачность** — мы всегда должны понимать, что делается, на каком этапе работа, какие результаты.
- **Результат** — цель не "попробовать разобраться", а полностью закрыть все проблемы.

**Коммуникация:**

- Регулярные отчеты о ходе работы
