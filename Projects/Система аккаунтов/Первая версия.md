# Система аккаунтов — Первая версия

## Ключевой принцип

Каждое юрлицо — **автономная ячейка**: независимая, масштабируемая и готовая к передаче или продаже без потери доступов. Аккаунты принадлежат компании, а не конкретным людям.

---

## 1. Каналы коммуникации

| Канал | Статус |
|-------|--------|
| Telegram | Активен |
| Почта (Яндекс / Gmail / Mail.ru) | Активна |
| MAX | Планируется |
| WhatsApp | Планируется |

---

## 2. Участники системы

### 2.1 Роли сотрудников

| Роль | Привязка к аккаунту | Особенности |
|------|---------------------|-------------|
| Главный администратор / Владелец | Персональная | Доступ к изолированному серверу |
| Генеральный директор (наёмный) | Персональная | Отдельные номера под каждый сервис |
| Руководитель направления | Через RDP | — |
| Линейный сотрудник | Через RDP | Бесшовная смена без перерегистрации |

### 2.2 Типы контрагентов

**Тип А — Договорные (оплата по безналу)**
Фулфилмент, специалисты по честному знаку и аналогичные.

**Тип Б — Без договора**
Дизайнеры, программисты, разовые подрядчики.

---

## 3. Типы аккаунтов

Система делится на два назначения: **внешние коммуникации** (с контрагентами) и **внутренние коммуникации** (внутри команды).

### 3.1 Внешние аккаунты — работа с контрагентами

Все внешние аккаунты доступны только через удалённый рабочий стол (RDP).

```
Сотрудник → RDP → Удалённый сервер → Telegram / Почта / Кабинет маркетплейса
```

#### Аккаунт №1 — Администраторский

| Параметр | Значение |
|----------|----------|
| Назначение | Создание всех чатов с подрядчиками |
| Доступ | Только главные лица компании |
| Расположение | Бухгалтерский компьютер ООО |
| Клиент | AyuGram (сохраняет удалённые сообщения и переписки) |
| Изоляция | Изолированный сервер, прямой внешний доступ закрыт |

#### Аккаунт №2 — Рабочий (внешний)

| Параметр | Значение |
|----------|----------|
| Назначение | Общение с подрядчиками и контрагентами |
| Доступ | Все сотрудники через RDP |
| Клиент | Стандартный Telegram-клиент |
| Добавление в чаты | Через пригласительные ссылки от Аккаунта №1 |

### 3.2 Внутренние аккаунты — общение внутри команды

| Параметр | Значение |
|----------|----------|
| Назначение | Коммуникация между сотрудниками |
| Доступ | Напрямую на устройствах сотрудников |
| Выдача | Аккаунты выдаются компанией, не личные |
| Устройство | Рабочее / личное устройство сотрудника |

---

## 4. Номера телефонов

- Все номера оформляются на **юридическое лицо**, не на физическое лицо
- **Гендиректор (наёмный):** отдельный номер под каждый сервис — банк, мессенджеры, почта; для чёткой фиксации действий
- **Линейные сотрудники:** один номер на роль; при смене сотрудника — номер и аккаунт остаются, новый сотрудник входит в тот же аккаунт

---

## 5. Безопасность

- **2FA (двухфакторная аутентификация)** — обязательна на всех аккаунтах; исключает вход с посторонних устройств
- **KeePassXC** — зашифрованное локальное хранилище:
  - Пароли и 2FA-коды всех аккаунтов
  - Сид-фразы

---

## 6. Масштабирование на другие юрлица

Каждое новое юрлицо = отдельная ячейка по единому шаблону:

1. Отдельные номера телефонов, оформленные на это юрлицо
2. Отдельный удалённый сервер
3. Аккаунт №1 (Администраторский) + Аккаунт №2 (Рабочий) для внешних коммуникаций
4. Внутренние аккаунты для команды
5. Отдельный раздел в KeePassXC для учётных данных
