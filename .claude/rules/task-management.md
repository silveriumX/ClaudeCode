# Task Management System

> ADHD-дружелюбная система задач с автоматическим контролем переносов

---

## ОБЯЗАТЕЛЬНАЯ ПРОВЕРКА ПРИ КАЖДОЙ СЕССИИ

При начале ЛЮБОЙ сессии AI ДОЛЖЕН:

1. **Проверить текущую дату** (из системного контекста)
2. **Найти последний план:** `Work/Задачи/План на *.md`
3. **Сравнить даты**

### Если дата изменилась:

```
AI: "Доброе утро! Сегодня {ТЕКУЩАЯ_ДАТА}.
Вижу план на {ДАТА_ПЛАНА} с {N} невыполненными задачами.

Перенести их на сегодня?
```

### Автоматический перенос:

1. Собрать АКТИВНЫЕ задачи (не "Ожидания")
2. Увеличить счётчики переносов
3. Выделить задачи с `[→3]` — требуют решения
4. Создать/обновить план на новый день
5. Сообщить результат

---

## Структура файлов

### `План на ДД.ММ.ГГГГ.md` — ежедневные планы

```markdown
# План на 25.01.2026

## Сделано
+1. Создать rules для проекта (создано 3 файла)

## Активные (MAX 5-7)
1. Главная задача дня
2. [→1] Задача с одним переносом
3. [→2] Задача с двумя переносами
4. [→3] Задача требует решения!

## Ожидания
- Ждать ответ от Дениса

## По времени
- 15:00 - Созвон
```

**ВАЖНО:**
- MAX 5-7 активных задач (реально выполнимо для ADHD)
- Ожидания отдельно (не давят)
- По времени — привязка к конкретному времени

---

## Система счётчиков переносов

### Формат метки: `[→N]`

```markdown
1. Передать экосистему Диме         # новая задача (без метки)
2. [→1] Передать экосистему Диме    # перенесена 1 раз
3. [→2] Передать экосистему Диме    # перенесена 2 раза
4. [→3] Передать экосистему Диме    # 3 раза = КРАСНАЯ ЗОНА!
```

---

## Правило 3-х переносов

Если задача переносилась **3 раза подряд** → AI ОБЯЗАН спросить:

```
Задача "{НАЗВАНИЕ}" переносилась 3 раза. Что делаем?

1. Сделать сегодня (высокий приоритет)
2. В "Когда-нибудь.md" (не горит, но важно)
3. Удалить (неактуально)
4. Делегировать (кому?)
```

---

## При выполнении задачи

1. Переместить в раздел "Сделано"
2. Добавить префикс `+`
3. Убрать метку переноса
4. Добавить краткий результат

---

## Перенос задач на новый день

### Алгоритм:

```
1. Найти план предыдущего дня
2. Собрать АКТИВНЫЕ задачи (НЕ "Ожидания")
3. Для каждой задачи:
   - Если нет метки → добавить [→1]
   - Если [→1] → изменить на [→2]
   - Если [→2] → изменить на [→3]
   - Если [→3] → СПРОСИТЬ РЕШЕНИЕ
   - Сохранять строку **ClickUp ID:** при переносе
4. Задачи "Ожидания" → перенести БЕЗ счётчика
5. Создать новый план
6. Сообщить результат
```

## Файл "Когда-нибудь.md"

Для задач которые:
- Важны, но НЕ горят
- Интересны, но нет времени
- Переносились 3+ раз и решили отложить

**Обзор:** Раз в неделю (воскресенье/понедельник)

---

## Еженедельный обзор

AI предлагает **раз в неделю**:

```
Еженедельный обзор:

Выполнено за неделю: {N} задач
Переносятся 3+ раз: {M} задач
В "Когда-нибудь": {K} задач

Провести чистку списков?
```

## Автоматическое обновление АКТУАЛЬНЫЕ_ЗАДАЧИ.md

При каждом изменении задач AI ОБЯЗАН обновлять `Work/Задачи/АКТУАЛЬНЫЕ_ЗАДАЧИ.md`:

### Когда обновлять:
- При добавлении/выполнении/переносе задачи
- При начале новой сессии (если дата изменилась)
- По запросу пользователя

### Что обновляется:
- Раздел "СЕГОДНЯ" — из актуального плана на сегодня
- Раздел "СРОЧНОЕ" — из плана "На этой неделе"
- Раздел "ДЕЛЕГИРОВАНО" — из Задачи.md + ожидания
- Раздел "НУЖНО УТОЧНИТЬ" — из плана
- Раздел "ПРОЕКТЫ" — из Задачи.md
- Раздел "ОЖИДАНИЯ" — из планов и Задачи.md
- Дата "Обновлено" — актуальная
- Счетчик "ИТОГО АКТИВНЫХ" — пересчитывается

## Запрещённые действия

AI НЕ ДОЛЖЕН:
- Игнорировать смену дня
- Переносить задачи без увеличения счётчика
- Копить задачи с `[→3]` без решения
- Смешивать "Активные" и "Ожидания"
- Добавлять > 7 активных задач без предупреждения
