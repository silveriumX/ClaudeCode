# Research System

Система ресерча мирового уровня.

## Когда использовать

- Пользователь просит провести исследование по теме
- Нужно собрать информацию из разных источников
- Нужен глубокий анализ какой-то темы
- Нужно найти информацию о людях, компаниях, технологиях
- Нужен структурированный отчёт

## Пошаговая инструкция

### Шаг 1: Определить параметры

1. **Тема исследования:** уточни тему и ключевые слова
2. **Глубина:** shallow / medium / deep
3. **Тип:**
   - Академическое (научные статьи)
   - Индустриальное (новости, блоги)
   - Техническое (документация, код)
   - О людях (LinkedIn, профили)
   - Комплексное (все типы)
4. **Предпочтения:** проверь `Projects/ResearchSystem/config/preferences.json`

### Шаг 2: Настроить источники

- EXA (приоритет 1) — семантический поиск
- Firecrawl (приоритет 2) — глубокий парсинг
- Для discovery: EXA
- Для глубокого анализа: EXA + Firecrawl

### Шаг 3: Выполнить поиск

1. Использовать EXA MCP или WebSearch для широкого поиска
2. Для параллельных исследований использовать Task tool с subagent_type="Explore"
3. Собрать результаты: источники, ключевые факты, противоречия

### Шаг 4: Синтезировать

1. Объединить информацию, выявить ключевые темы
2. Найти общие паттерны, определить пробелы
3. Сравнить информацию из разных источников, отметить расхождения

### Шаг 5: Сгенерировать отчёт

**Структура:**

```markdown
# Research Report: {Topic}

**Date:** {date}
**Sources:** {count}

## Executive Summary
{Краткое резюме}

## Key Findings
{Ключевые находки}

## Detailed Analysis
{Детальный анализ}

## Sources
{Список источников}

## Next Steps
{Рекомендации}
```

**Сохранить:** `Projects/ResearchSystem/data/reports/report_YYYYMMDD_HHMMSS.md`

- [ ] Результаты синтезированы
- [ ] Предпочтения применены
- [ ] Отчёт структурирован
- [ ] Источники указаны
- [ ] Отчёт сохранён

## Что НЕ делать

- Не игнорируй предпочтения пользователя
- Не используй только один источник (минимум 3-5)
- Не генерируй отчёт без синтеза
- Не делай слишком длинные отчёты без структуры
