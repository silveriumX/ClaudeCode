# Daily Plan Management

Автоматическое создание и управление ежедневными планами с учётом системы переносов задач.

## Основной workflow

### 1. Проверка текущего состояния

```
1. Получить текущую дату из системного контекста
2. Найти последний план: Work/Задачи/План на *.md
3. Извлечь дату из имени файла
4. Если даты отличаются — предложить перенос
```

### 2. Создание нового плана

**Структура файла:**

```markdown
# План на ДД.ММ.ГГГГ (день недели)

## Сделано
*(пока пусто)*

## Активные (MAX 5-7) — приоритет на день
1. [метка переноса] Задача 1
2. Задача 2

## Дедлайн
- [ ] Задача с дедлайном

## Главное на сегодня
### Категория 1
- [ ] Задача

## Ожидания
- Что ждём от кого

## По времени
- **Время** — событие

## Заметки (рефлексия)
- Мысли и заметки дня
```

### 3. Перенос задач с предыдущего плана

**Правила переноса:**

```
def перенести_задачу(задача, секция):
    if секция == "Ожидания":
        # Ожидания переносятся БЕЗ счётчика
        return задача

    if секция == "Активные":
        if "[→" in задача:
            count = извлечь_счётчик(задача) + 1
            if count >= 3:
                СПРОСИТЬ_РЕШЕНИЕ(задача)  # СТОП! Ждать ответа.
            return обновить_счётчик(задача, count)
        else:
            return добавить_метку(задача, "[→1]")

    return задача
```

**Метки переносов:**
- Без метки — новая задача
- `[→1]` — перенесена 1 раз
- `[→2]` — перенесена 2 раза
- `[→3]` — перенесена 3 раза, **требует решения!**

### 4. Обработка задач с [→3]

AI ОБЯЗАН спросить:

```
Задача "{название}" переносилась 3 раза подряд.

Что делаем:
1. Сделать сегодня обязательно
2. В "Когда-нибудь.md" (важно, но не горит)
3. Удалить (неактуально)
4. Делегировать (указать кому)
```

**НЕ переносить задачу пока не получен ответ.**

### 5. Секции которые переносятся

**Переносить:**
- Активные (MAX 5-7) — с увеличением счётчика
- Дедлайн — если не выполнено
- Главное на сегодня — актуальные подзадачи
- Ожидания — БЕЗ счётчика

**НЕ переносить:**
- Сделано — это история предыдущего дня
- Заметки — рефлексия конкретного дня

### 6. Сохранение ClickUp ID

**КРИТИЧНО:** При переносе СОХРАНЯЙ строку `ClickUp ID:` если она есть в задаче:

```markdown
# Было в плане на 09.02
1. [→1] Передать документы Диме
   ClickUp ID: abc123xyz

# Должно быть в плане на 11.02
1. [→2] Передать документы Диме
   ClickUp ID: abc123xyz  # <-- НЕ ПОТЕРЯТЬ!
```

### 7. Автоматическое обновление АКТУАЛЬНЫЕ_ЗАДАЧИ.md

После создания плана обновить `Work/Задачи/АКТУАЛЬНЫЕ_ЗАДАЧИ.md`:
- Раздел "СЕГОДНЯ" — из актуального плана
- Раздел "СРОЧНОЕ" — из плана "На этой неделе"
- Раздел "ДЕЛЕГИРОВАНО" — из ожиданий
- Дата "Обновлено" — актуальная
- Счетчик "ИТОГО АКТИВНЫХ" — пересчитать
