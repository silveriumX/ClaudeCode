# HN Search

> Семантический поиск по Hacker News через Exa MCP.
> HN — лучший источник для tech-инсайтов от практикующих инженеров и фаундеров.

---

## Когда использовать

- "Что думают на HN о X?"
- "Есть ли обсуждение Y на Hacker News?"
- "Что инженеры говорят про [технологию]?"
- "Show HN проекты по теме Z"
- "/hn-search"

---

## Почему HN особенный

Hacker News — это не просто форум. Это **сигнальная среда** для раннего обнаружения трендов:
- Комментаторы — Senior Engineers, CTOs, фаундеры
- Высокий SNR (мало маркетинга, много опыта)
- Ask HN, Show HN, Tell HN — уникальные форматы
- Обсуждения технических деталей недоступных в других местах

---

## Алгоритм (через Exa MCP)

### Основные поисковые паттерны

```
# Общее обсуждение технологии
mcp__exa__web_search_exa(
  query="[тема] discussion opinions",
  include_domains=["news.ycombinator.com"]
)

# Show HN — что люди запускают
mcp__exa__web_search_exa(
  query="Show HN [тема]",
  include_domains=["news.ycombinator.com"]
)

# Ask HN — что люди спрашивают
mcp__exa__web_search_exa(
  query="Ask HN [тема]",
  include_domains=["news.ycombinator.com"]
)

# Критика и проблемы
mcp__exa__web_search_exa(
  query="[тема] problems issues criticism HN",
  include_domains=["news.ycombinator.com"]
)
```

### Параллельные поиски для полного охвата

```
Поиск 1: обсуждения темы (general)
Поиск 2: "Show HN" + тема (что запускают)
Поиск 3: "Ask HN" + тема (что спрашивают)
Поиск 4: критика / проблемы
```

---

## Ограничения

- HN хорошо индексируется Exa — обычно хорошее покрытие
- Самые горячие треды могут быть недоступны через `include_domains` — тогда дополнительно искать через `hn.algolia.com`
- Комментарии в самих тредах Exa не всегда захватывает — для глубокого прочтения использовать `WebFetch` на конкретный URL
